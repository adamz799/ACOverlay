# AC Overlay - 神力科莎遥测数据显示插件

一个实时显示 Assetto Corsa (神力科莎) 遥测数据的透明悬浮窗口插件，支持数据录制和 RaceChrono 兼容格式导出。

## 功能特性

### 📊 实时数据监测
- 油门/刹车输入曲线图（历史轨迹，宽幅显示）
- 油门/刹车垂直进度条
- 当前挡位和速度
- 转速指示灯（RPM）
- G力显示（横向/纵向/总G值数字显示）
- 方向盘转向角度

### 🎮 智能显隐
- 自动检测驾驶状态，开始驾驶时显示，停止时隐藏
- 手动控制可覆盖自动逻辑

### 📝 遥测数据录制
- **自动录制**: 驾驶时自动开始录制
- **手动控制**: F11 手动开始/停止录制
- **RaceChrono 兼容**: CSV 格式可直接导入 RaceChrono 分析
- **文件命名**: `时间_赛道_车辆.csv` 格式，便于管理
- **优化存储**: 25Hz 采样率，智能精度控制，大幅减小文件体积

### 记录的数据字段 (80+项)
| 类别 | 字段 | 精度 |
|------|------|------|
| **时间** | 时间戳、圈数、圈时间 | 3位小数 |
| **输入** | 油门、刹车、离合、方向盘角度 | 2位小数 |
| **运动** | 速度(km/h, m/s)、挡位、转速 | 2-3位小数 |
| **速度向量** | X/Y/Z 三轴速度 | 3位小数 |
| **G力** | 横向、纵向、垂直加速度 | 3位小数 |
| **姿态** | 航向角、俯仰角、横滚角 | 2位小数 |
| **角速度** | X/Y/Z 三轴角速度 | 3位小数 |
| **位置** | 赛道位置、行驶距离、世界坐标 | 2-4位小数 |
| **轮胎** | 滑动、负载、胎压、温度、磨损 (4轮) | 0-3位小数 |
| **刹车** | 刹车温度 (4轮) | 整数 |
| **悬挂** | 悬挂行程、车身高度 | 2位小数 |
| **燃油** | 剩余燃油、每圈油耗 | 2-3位小数 |
| **动力** | 涡轮增压、ERS/KERS 能量 | 1-2位小数 |
| **辅助** | TC、ABS、刹车前后比、发动机制动 | 1位小数 |
| **DRS** | 可用状态、启用状态 | - |
| **损伤** | 前/后/左/右/中 损伤程度 | - |
| **状态** | 进站状态、限速器、扇区、出界轮胎数 | - |
| **环境** | 气温、路温、风速、风向 | 0-1位小数 |
| **比赛** | 排行、旗帜状态 | - |

### ⌨️ 快捷控制
| 快捷键 | 功能 |
|--------|------|
| `F9` | 切换显示/隐藏 |
| `F10` | 切换自动显隐功能 |
| `F11` | 开始/停止录制 |
| `F12` | 切换自动录制功能 |

### 🖼️ 界面特性
- 无边框透明窗口，浮在游戏画面上层
- 支持拖拽移动位置
- 连接状态实时指示
- 录制状态指示（红色闪烁）
- 系统托盘菜单控制

## 安装

### 方式一：直接使用可执行文件

下载 `ACOverlay.exe` (约 25MB)，双击运行即可。

### 方式二：从源码运行

#### 1. 安装 Python 3.10+

确保已安装 Python 3.10 或更高版本。

#### 2. 安装依赖

```bash
pip install -r requirements.txt
```

#### 3. 运行程序

```bash
python main.py
```

## 使用说明

### 基本使用
1. 启动 Assetto Corsa 游戏
2. 运行本程序
3. 程序会自动连接到游戏的共享内存接口
4. 连接状态指示灯：
   - 🟢 绿色 = 已连接
   - 🔴 红色 = 未连接/等待连接

### 数据录制
1. **自动录制** (默认开启): 检测到驾驶时自动开始录制
2. **手动录制**: 按 `F11` 开始/停止录制
3. **日志位置**: 程序目录下的 `logs` 文件夹
4. **文件格式**: `20240101_120000_spa_ferrari_488.csv`

### 系统托盘
- 右键点击托盘图标打开菜单
- 双击托盘图标切换显示状态
- 可快速访问日志文件夹

## 日志文件格式

生成的 CSV 文件兼容 RaceChrono，包含以下特性：
- 文件头包含会话元数据（车辆、赛道、时间等）
- 标准化的列名，可直接导入分析工具
- 智能精度控制（关键数据高精度，次要数据低精度）
- 25Hz 采样率（每秒 25 个数据点）

### 数据精度说明

为了平衡分析精度和文件大小，不同字段采用不同的小数位数：

| 数据类型 | 精度 | 说明 |
|----------|------|------|
| 速度、输入、方向盘 | 2位 | 满足精细对比需求 |
| G力、速度分量、轮胎滑动 | 3位 | 高精度分析 |
| 位置坐标 | 2位 | 轨迹重建足够 |
| 赛道位置 | 4位 | 精确的赛道进度 |
| 温度、胎压 | 0-1位 | 变化较慢的数据 |

## 在 RaceChrono 中使用

1. 将 CSV 文件传输到手机/平板
2. 打开 RaceChrono Pro
3. 选择 "导入会话"
4. 选择生成的 CSV 文件
5. 即可查看圈速分析、轨迹对比等

## 技术说明

本程序通过读取 Assetto Corsa 的共享内存接口获取遥测数据：

- `acpmf_physics` - 物理数据（油门、刹车、速度、G力等）
- `acpmf_graphics` - 图形数据（游戏状态、位置等）
- `acpmf_static` - 静态数据（车辆、赛道信息等）

### 性能优化
- **UI 刷新**: 约 60fps，确保流畅显示
- **数据采样**: 25Hz，平衡精度与存储
- **EXE 体积**: 约 25MB（排除未使用的 Qt 模块）

## 已知问题

- 需要管理员权限运行才能注册全局热键
- 仅支持 Windows 系统
- 需要先启动游戏再运行本程序，或等待自动重连
- 世界坐标基于游戏内部坐标系，非真实 GPS
